Virtualisation de Moodle

Hôte apparent de l'instance

Ce champ permet de définir le nom Web apparent du nouveau site. En général, ce nom doit comporter une métabalise <%%INSTANCE%%> qui permet la différenciation de chaque nouvelle adresse Web pour chacune des instances. Le nom court est utilisé (ramené en minuscules) pour la constitution de ce sous-domaine.

Si la différentiation se fait sur un sous-domaine (exemple : %%INSTANCE%%.mondomaine.org), la stratégie de fabrication des instances se base sur un des sous domaines virtualisés du serveur Apache, conduisant au même DocumentRoot.

Une configuration typique pour Apache dans ce cas de figure est :

<VirtualHost 127.0.0.1>
    ServerAdmin admin@foo.org
    ServerName default.mondomaine.org
    ServerAlias *.mondomaine.org
	VirtualDocumentRoot "chemin_vers_défaut/vmoodle_defaut"
    ErrorLog logs/vmoodle-error_log
    CustomLog logs/vmoodle-access_log common
</VirtualHost>

D'autres méthodes sont probablement possibles, mais n'ont pas fait l'objet d'évaluations.
